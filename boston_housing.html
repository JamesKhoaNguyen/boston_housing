Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 always fails to predict the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is no better than one that naively predicts the mean of the target variable.

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer:The benefit to splitting a dataset into some ratio of trainging and testing subsets for a learning algorithm is to reduce the possibility of overfitting one's model to the data. If 100% of your gathered data is used as training data and then the model was tested on that same training data, the testing score will likely be high but would be biased. The reason it would be biased is because the model is being rewarded to recall data it has seen before rather than make a prediction on new never before seen data. If only a portion of the data is used to train, then the model will encounter inputs it has never seen before which will force it to generalize and predict. Of course, even if you gave it 100% of your data to train on the first time around, you could always get more never before seen data to use as a training set but that requires time. By randomly dividing up your data into different combinations of testing and training data, you're avoiding overfitting and saving time by being able to reuse your data.

Question 7 - Grid Search¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer: Machine Learning algorithms typically have hyperparameters or parameters that are not features of the immediate problem you're trying to solve but are components of the specific algorithm used. For example, when the regression learning algorithm, one attempts to minimize the cost function outputs and in doing so one must set the learning rate. This learning rate is considered a hyperparameter since it's not one of the features like RM, PTRATIO, or LSTAT of the problem we're trying to solve. In theory, the grid search algorithm finds the cartesian products of your hyperparameters, assuming each hyperparameter is a set of some values. This can be used to optimize a learning algorithm by defining sets of values for each hyperparameter manually, since you aren't sure which combination of values of hyperparameters produces the optimum model, and then using grid search to efficiently look through the search space and apply different combinations as well as report the results for each one. Why do it manually when you can use grid search?

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: The k-fold cross-validation training technique basically splits your data into k sets/bins/folds and k-1 sets are designated as training data while the 1 set becomes your testing data. The point of using grid search is to optimize the hyperparameters but it doesn't indicate whether that model is underfitting or overfitting. Cross-validation can optimize the fit of your model in parallel with the optimizing the hyperparameter by ensuring that the testing data is not the same as training data. Additionally, k-fold cross validation also allows your training and testing data to be randomized in case the ordering of the data introduces biases when doing test-train splitting. In summary, k-fold validation ensures eery data point is used in the training set at aleas once and this guarantees that you've had the model make a prediction on all possible gathered inputs. This ensures the optimal set of hyperparameters is chosen based on the available data points. The choice of the optimal set of hyperparameters are furthered by the fact that the selection process is based off the averaged results of the k-fold cross-validation. By using the average, the robustness of the model is ensured.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV
from sklearn.metrics import make_scorer

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':list(range(1,11))}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric,greater_is_better = True)
    # TODO: Create the grid search object
    grid = GridSearchCV(regressor,params,scoring=scoring_fnc,verbose=True,cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

Question 11 - Applicability¶

In a few sentences, discuss whether the constructed model should or should not be used in a real-world setting.
Hint: Some questions to answering:

  • How relevant today is data that was collected from 1978?
  • Are the features present in the data sufficient to describe a home?
  • Is the model robust enough to make consistent predictions?
  • Would data collected in an urban city like Boston be applicable in a rural city?

Answer: Regardless of what the actual value of money was back then, people still value features such as the poverty level, the amount of time a student is able to interact with a teacher, and amount of rooms which usually relates to square footage. In terms of determining relative value between homes with these features, the data from 1978 should be just as relevant as today.The features present in the data are sufficient to an extent, but people also choose homes based on how the interior and exterior aesthetics and that feature is difficult to define. Since the aesthetic feature of a home was not incorporated, these features LSTAT,RM, and PTRATIO alone are not sufficient to describe a home. That's not to say that those 3 features are not sufficient to give us an idea of how much a particular home may cost. Assuming that vs.PredictTrials() fucntion is reporting the predicted prices for the one same client, then the model is fairly robust since the standard deviation between the predictions is ~22,533 dollars which is a good ball-park estimate relative to the price of homes being in the six-figures range. The data collected in an urban city like Boston wouldn't be applicable in a rural city since what people look for would be different which is to say different features need to be considered in our model. For example, in rural cities, the PTRATIO would be less of a factor since schools would be smaller in general. In summary, this model can be used in a real-world setting if the accurate data about PTRATIO and LSTAT were readily available but just from a quick search on Google, this information hasn't been standardized/formalized enough to be reliable. The only data that is easily available is the prices of the homes and the amount of rooms and the names of the neighborhoods. But if the accuracy of these parameters can't be ensured, then this model shouldn't be used.
